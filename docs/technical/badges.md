# Badge System Documentation

This document explains how the README badge system works, how to update badges, and how to troubleshoot issues.

## Overview

Mapper uses a combination of static and dynamic badges at the top of the README:

```markdown
![Version](https://img.shields.io/badge/version-0.1.2-blue.svg)
![Tests](https://img.shields.io/endpoint?url=https://gist...)
![Coverage](https://img.shields.io/endpoint?url=https://gist...)
![Python](https://img.shields.io/badge/python-3.10%2B-blue.svg)
```

## Badge Types

### 1. Version Badge (Static, Auto-Updated)
- **Type**: Static badge generated by shields.io
- **Update Method**: Automatically updated by `bump-my-version`
- **URL Format**: `https://img.shields.io/badge/version-{VERSION}-blue.svg`
- **Color**: Always blue

**How it works:**
- When you run `just version patch|minor|major`, bump-my-version updates `pyproject.toml`, `src/mapper/__init__.py`, `CHANGELOG.md`, and the version badge in `README.md`
- The version string in the badge URL is directly replaced (e.g., `version-0.1.1-blue` → `version-0.1.2-blue`)

**Manual update (if needed):**
```bash
# Edit README.md and change the version number in the badge URL
![Version](https://img.shields.io/badge/version-0.1.2-blue.svg)
#                                            ^^^^^ change this
```

### 2. Tests Badge (Dynamic, Auto-Updated by CI)
- **Type**: Dynamic badge using shields.io endpoint + GitHub Gist
- **Update Method**: Automatically updated by CI on push to main
- **Gist URL**: https://gist.github.com/octo-youcef/9501806ed5eac873dd324bc606c6dd79
- **Gist File**: `mapper-tests.json`
- **Color**: Always green ("success")

**How it works:**
1. CI workflow (`.github/workflows/update-coverage-badge.yml`) runs on push to main
2. Runs pytest and extracts test count from output (e.g., "45 passed")
3. Creates JSON file with shields.io endpoint format:
   ```json
   {
     "schemaVersion": 1,
     "label": "tests",
     "message": "45 passing",
     "color": "success"
   }
   ```
4. Updates the `mapper-tests.json` file in the gist via GitHub API
5. shields.io reads from the gist and renders the badge
6. Badge updates within ~5 minutes (shields.io caching)

**Manual update (if needed):**
```bash
# Update the gist file manually at:
# https://gist.github.com/octo-youcef/9501806ed5eac873dd324bc606c6dd79

# Or via API:
curl -X PATCH \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: application/vnd.github+json" \
  "https://api.github.com/gists/9501806ed5eac873dd324bc606c6dd79" \
  -d '{
    "files": {
      "mapper-tests.json": {
        "content": "{\n  \"schemaVersion\": 1,\n  \"label\": \"tests\",\n  \"message\": \"50 passing\",\n  \"color\": \"success\"\n}\n"
      }
    }
  }'
```

### 3. Coverage Badge (Dynamic, Auto-Updated by CI)
- **Type**: Dynamic badge using shields.io endpoint + GitHub Gist
- **Update Method**: Automatically updated by CI on push to main
- **Gist URL**: https://gist.github.com/octo-youcef/9501806ed5eac873dd324bc606c6dd79
- **Gist File**: `mapper-coverage.json`
- **Color**: Dynamic based on coverage percentage

**Color mapping:**
- ≥90%: `brightgreen`
- 75-89%: `green`
- 60-74%: `yellow`
- 40-59%: `orange`
- <40%: `red`

**How it works:**
1. CI workflow runs on push to main
2. Runs pytest with coverage and extracts percentage from `coverage.json`
3. Determines badge color based on coverage percentage
4. Creates JSON file:
   ```json
   {
     "schemaVersion": 1,
     "label": "coverage",
     "message": "84%",
     "color": "brightgreen"
   }
   ```
5. Updates the `mapper-coverage.json` file in the gist via GitHub API
6. shields.io reads from the gist and renders the badge

**Manual update (if needed):**
```bash
# Update the gist file manually or via API (similar to tests badge)
```

### 4. Python Badge (Static, Manual)
- **Type**: Static badge
- **Update Method**: Manual edit of README.md
- **URL Format**: `https://img.shields.io/badge/python-3.10%2B-blue.svg`
- **Color**: Always blue

**How to update:**
```bash
# Edit README.md when Python version requirements change
![Python](https://img.shields.io/badge/python-3.10%2B-blue.svg)
#                                          ^^^^^^ change this
```

## CI Workflow

The `.github/workflows/update-coverage-badge.yml` workflow handles automatic badge updates.

**Trigger**: Push to `main` branch

**Steps:**
1. Checkout code
2. Install dependencies
3. Run tests with coverage
4. Extract test count and coverage percentage
5. Determine coverage badge color
6. Create JSON files for both badges
7. Update gist via GitHub API

**Required Secrets:**
- `GIST_TOKEN`: GitHub Personal Access Token with `gist` scope

## Troubleshooting

### Badge not updating after merge

**Symptom**: Badge still shows old value after PR merges to main

**Possible causes:**
1. **shields.io caching** - Badges cache for ~5 minutes
   - Wait 5-10 minutes and hard refresh the page (Ctrl+Shift+R)

2. **CI workflow failed** - Check the workflow run
   ```bash
   gh run list --workflow=update-coverage-badge.yml --limit=5
   gh run view <run-id> --log
   ```

3. **GIST_TOKEN secret missing or invalid**
   - Check: https://github.com/octo-youcef/mapper/settings/secrets/actions
   - Verify token has `gist` scope
   - Regenerate if needed: https://github.com/settings/tokens

4. **Gist ID changed** - Verify gist ID in workflow matches actual gist
   - Gist: https://gist.github.com/octo-youcef/9501806ed5eac873dd324bc606c6dd79
   - Workflow: `.github/workflows/update-coverage-badge.yml` line 58

### CI workflow failing

**Check the logs:**
```bash
gh run list --workflow=update-coverage-badge.yml --limit=1
gh run view --log
```

**Common issues:**
- Missing `GIST_TOKEN` secret
- Invalid token or revoked permissions
- Gist ID mismatch
- pytest output format changed (breaking regex)

### Badge showing "invalid"

**Symptom**: Badge displays "invalid" instead of data

**Causes:**
1. **Malformed JSON in gist** - Check gist file syntax
2. **Wrong schemaVersion** - Should be `1`
3. **Missing required fields** - Must have `schemaVersion`, `label`, `message`, `color`

**Fix:**
Edit the gist file manually to ensure valid JSON:
```json
{
  "schemaVersion": 1,
  "label": "tests",
  "message": "45 passing",
  "color": "success"
}
```

### Version badge not updating with bump-my-version

**Check configuration:**
```toml
# pyproject.toml
[[tool.bumpversion.files]]
filename = "README.md"
search = 'version-{current_version}-blue'
replace = 'version-{new_version}-blue'
```

**Verify the search pattern matches README:**
```bash
grep "version-" README.md
# Should show: ![Version](https://img.shields.io/badge/version-0.1.2-blue.svg)
```

## Adding a New Badge

To add a new dynamic badge:

1. **Add file to gist:**
   ```bash
   # Create JSON file
   cat > new-badge.json << EOF
   {
     "schemaVersion": 1,
     "label": "new metric",
     "message": "value",
     "color": "blue"
   }
   EOF

   # Add to gist
   gh api -X PATCH /gists/9501806ed5eac873dd324bc606c6dd79 \
     -f files[mapper-new-badge.json][content]=@new-badge.json
   ```

2. **Update CI workflow** to generate and upload the new badge data

3. **Add badge to README:**
   ```markdown
   ![New Badge](https://img.shields.io/endpoint?url=https://gist.githubusercontent.com/octo-youcef/9501806ed5eac873dd324bc606c6dd79/raw/mapper-new-badge.json)
   ```
